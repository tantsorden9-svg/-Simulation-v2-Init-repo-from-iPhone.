# Simulation v2 — core

Це репозиторій ядра системи **Simulation v2**.

Тут зібрані:
- `PHYSICS.md` — глобальні закони системи;
- `REGISTRY.md` — реєстр усіх SIM-ів;
- базові шаблони та інфраструктура для створення нових симуляцій.

## Структура репозиторію (v0.1)

- `README.md` — цей файл.
- `PHYSICS.md` — опис законів "фізики" системи.
- `REGISTRY.md` — список SIM-ів та їхній статус.
- `sim/` (якщо зʼявиться) — папки окремих SIM-ів.

## Що таке SIM
**SIM-XXX** — це окрема симуляція / сценарій, що живе всередині загальної фізики:
- може мати свою специфіку та перевизначення;
- використовує загальні принципи з `PHYSICS.md`;
- описана в `REGISTRY.md`.

## Як створити новий SIM (чернетково)
1. Взяти шаблон (коли зʼявиться) або скопіювати **SIM-000** як базу.
2. Створити власну папку, наприклад: `sim/sim-001-my-experiment/`.
3. Описати:
   - який це сценарій;
   - які сутності він має;
   - як для нього виглядає tick.
4. Додати запис у `REGISTRY.md`.

## Версіонування
- Основні зміни глобальної фізики — в `PHYSICS.md` з оновленням версії.
- Кожен SIM привʼязаний до конкретної версії PHYSICS.
- Репозиторій еволюціонує так, щоб можна було:
  - відтворити старі SIM-и з їхніми версіями фізики;
  - запускати нові на актуальних правилах.

## Статус проекту
Ранній етап, експериментальна стадія:
- формуємо ядро фізики;
- реєструємо перші SIM-и;
- узгоджуємо структуру папок і шаблонів.

Ціль — мати мінімальне, але чітке ядро, поверх якого легко будувати ігрові, наукові чи концептуальні симуляції.
