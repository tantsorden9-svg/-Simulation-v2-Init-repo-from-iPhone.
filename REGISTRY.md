# REGISTRY.md
Версія: 0.1

## Призначення
REGISTRY містить список усіх SIM-* у цьому репозиторії та їхній статус.

## Поля реєстру
- **ID** — унікальний ідентифікатор, формат `SIM-000`, `SIM-001`, ...
- **Path** — шлях до папки SIM у репозиторії або `.` якщо в корені.
- **Physics** — версія PHYSICS, з якою сумісна симуляція.
- **Status** — `draft` | `active` | `archived`.
- **Role** — для чого SIM (core, експеримент, демо, утиліта).
- **Summary** — короткий опис (1–2 речення).

## Реєстр SIM-ів

| ID      | Path              | Physics | Status  | Role  | Summary                                   |
|--------:|-------------------|---------|---------|-------|-------------------------------------------|
| SIM-000 | `.`               | 0.1     | active  | core  | Базове ядро системи та тестовий полігон.  |
| SIM-001 | `sim/sim-001_core/`   | 0.1 | draft  | lab  | Перший експериментальний сценарій. |
| SIM-002 | `sim/sim-002_world/`  | 0.1 | draft  | lab  | Другий сценарій для перевірки ідей. |

## Правила додавання нового SIM
1. Створити папку / структуру для SIM (`sim/SIM-00X-*` або іншу прийняту схему).
2. Додати рядок у таблицю, проставити:
   - новий ID;
   - коректний `Path`;
   - версію `Physics`;
   - `Status = draft`.
3. Коли SIM стає стабільним — змінити `Status` на `active`.
4. Старі або невдалі SIM-імплементації — `archived`.
